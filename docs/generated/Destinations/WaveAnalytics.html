Wave Analytics

The Wave Analytics destination writes data to Salesforce Wave Analytics. The destination connects to Wave Analytics to create a dataset with external data.

When you configure the Wave Analytics destination, you define connection information and specify the edgemart alias or name of the dataset to create.

The destination typically creates multiple datasets for the edgemart alias, based on the dataset wait time that you configure for the destination. After no data has arrived for the specified wait time, the destination closes and uploads the dataset to Wave Analytics. When new data arrives, the destination creates a new dataset. Each dataset is named with the edgemart alias name and appended with the timestamp of when the dataset is uploaded.

You can optionally configure the destination to use a Wave Analytics dataflow to combine multiple datasets together. You can also specify the metadata of the external data in a JSON format.

Dataflow

The Wave Analytics destination typically creates multiple datasets, based on the configured dataset wait time. You can optionally configure the destination to use a Wave Analytics dataflow to combine multiple datasets together.

A Wave Analytics dataflow includes instructions and transformations to combine datasets. Create the dataflow in Wave Analytics. Then when you configure the Wave Analytics destination, specify the name of the existing dataflow. The dataflow should not contain any content, as the Wave destination overwrites any existing content.

By default, a dataflow runs every 24 hours. However, you can configure the dataflow to run each time the destination closes and uploads a dataset to Wave Analytics. In Wave Analytics, you can run a dataflow a maximum of 24 times in a 24 hour period. So if you choose to run the dataflow after each dataset upload, make sure that the configured dataset wait time is more than an hour.

For more information about creating dataflows, see the Salesforce Wave Analytics documentation.

Configuring a Wave Analytics Destination

Configure a Wave Analytics destination to write data to Salesforce Wave Analytics.
  1. In the Properties panel, on the General tab, configure the following properties:
    General Property Description
    Name Stage name.
    Description Optional description.
    Required Fields Fields that must include data to be passed into the stage.
    Tip: You might include fields that the stage uses.
    Preconditions Conditions that must evaluate to TRUE to allow a record to enter the stage for processing. Click Add to create additional preconditions.
    On Record Error Error record handling for the stage:
    • Discard - Discards the record.
    • Send to Error - Sends the record to the pipeline for error handling.
    • Stop Pipeline - Stops the pipeline.
  2. On the Wave tab, configure the following properties:
    Wave Property Description
    Username Salesforce username in the following email format: <text>@<text>.com.
    Password Salesforce password.
    Tip: To secure sensitive information such as usernames and passwords, you can use runtime resources or Hashicorp Vault secrets. For more information, see Using Runtime Resources or Accessing Hashicorp Vault Secrets.
    Edgemart Alias Dataset name. The alias must be unique across an organization.

    Each dataset is named with the edgemart alias and appended with the timestamp of when the dataset is uploaded. If you use a dataflow to combine multiple datasets, the destination uses the edgemart alias as the name of the aggregated dataset.
    Dataset Wait Time (secs) Maximum time in seconds to wait for new data to arrive. After no data has arrived in this amount of time, the destination closes and uploads the dataset to Wave Analytics. When new data arrives, the destination creates a new dataset.

    The dataset wait time must be at least as long as the Batch Wait Time for the origin in the pipeline.
    Use Dataflow Determines whether to use a Wave Analytics dataflow to combine multiple datasets together.
    Dataflow Name Name of the existing dataflow.

    You must create the dataflow in Wave Analytics.

    Run Dataflow After Upload Determines whether the destination runs the dataflow each time that it uploads a dataset to Wave Analytics.
    Metadata JSON Metadata in JSON format that describes the schema of the dataset.

    Enter metadata so that Wave Analytics can correctly interpret the datatype of the data. If you do not enter metadata, then Wave Analytics treats every field as text.

    For more information about how Wave Analytics handles JSON metadata for uploaded external data, see the Salesforce Wave Analytics developer documentation.
